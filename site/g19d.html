G19D

Summary

G19D is a daemon process that must run as root, and provides low level access to the G19 keyboard for Gnome15 through a local TCP/IP socket. It is mostly based on MultiCoreNop's' Logitech-G19-Linux-Daemon which is available from here, but with a few additions and bug-fixes.

It is intended as a temporary solution until there is a more efficient method of communicating with the keyboard. There is kernel patch here, but I can find no other information about it.

Pre-requisites

You will need all of these to compile or run G19D. If you use a package for your distribution, these will be downloaded for you. If you wish to compile from source, you will have to obtain and install them yourself.

  • Python - Programming language. G19D was developed with Python 2.6
  • Python Image Library - Used for the splash screen.
  • PyUSB - Python USB bindings.
  • Python UINPUT - Python UINPUT bindings. Used to fire multimedia key events. This is the only library that is not available in the official Ubuntu repositories, although there is a PPA available for it here, or you can get it from the Gnome15 PPA.

Source

Download the source tarball g19d-0.0.4.tar.gz and follow the usual source package installation procedure.

./configure --prefix=/usr
make
sudo make install

This procedure will not install any init scripts, so if you want to run g19d at boot, you will have to edit the appropriate scripts for your distribution.

Packages

Ubuntu 10.10 and 10.4 packages are available in the Gnome15 PPA.

First, add the PPA to your sources. You may do this in a number of ways (Synaptic etc), or from the command line:-

sudo add-apt-repository ppa:tanktarta/gnome15
sudo apt-get update

Now install g19d.

sudo apt-get install g19d

You will also have to enable the daemon before starting it.

  1. Open /etc/default/g19d in your favourite text editor.
  2. Uncomment the G19D_ENABLED line and ensure it is set to 'y'
  3. Save the file.
  4. Start the daemon :- 
    sudo service g19d start
